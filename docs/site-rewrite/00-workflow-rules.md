# Workflow Rules

Этот документ объясняет, как заполнять черновики так, чтобы любая нейросеть или человек потом могли на их основе внести изменения в сайт.

## Главный принцип

Пишем не абстрактные идеи, а конкретные инструкции для будущей правки.

Плохо:
- "Сделать страницу лучше"
- "Переписать блок"
- "Обновить кейсы"

Хорошо:
- "На первом экране заменить заголовок на: ..."
- "Удалить блок 'Мы подходим не всем'"
- "Добавить новый блок после блока услуг с заголовком: ..."
- "Изменить CTA кнопки с 'Обсудить проект' на 'Получить аудит'"

## Формат записи

В каждом файле страницы по возможности заполняем:
- `Статус`
- `Цель страницы`
- `Что оставить`
- `Что изменить`
- `Что удалить`
- `Что добавить`
- `Новый текст`
- `SEO`
- `CTA`
- `Вопросы / TODO`

## Как писать тексты

Если текст уже утверждён, вставляем его целиком.

Пример:

```md
## Новый текст

### Hero
Заголовок: Комплексный маркетинг для производственных компаний

Подзаголовок:
Помогаем производителям и B2B-компаниям получать заявки через сайт, SEO, CRM и аналитику.

Кнопка:
Получить план роста
```

Если текст не утверждён, фиксируем задачу максимально предметно.

Пример:

```md
## Что изменить
- Переписать первый экран под производственные компании.
- Убрать акцент на "не всем подходим", сделать более дружелюбительный оффер.
- Добавить конкретику про отрасли: производство, металлопрокат, B2B-услуги.
```

## Правило для будущей нейросети

Во всех файлах страницы нужно писать так, будто это инструкция на изменение сайта:
- где находится блок;
- что с ним сделать;
- какой текст поставить;
- какой смысл должен остаться;
- чего нельзя потерять.

## Что не смешивать

Не смешиваем в одном абзаце:
- мысли о бренде;
- тексты для страницы;
- технические задачи;
- спорные идеи.

Для этого:
- общие договорённости пишем в `01-global-content.md`;
- решения фиксируем в `02-change-log.md`;
- правки по конкретной странице ведём в файле этой страницы.

## Статусы

Используем один из статусов:
- `draft`
- `in_progress`
- `review`
- `approved`
- `ready_for_implementation`

## Если есть несколько вариантов

Пишем так:

```md
## Варианты

### Вариант A
...

### Вариант B
...

Рекомендация:
Выбрать вариант A, потому что ...
```
